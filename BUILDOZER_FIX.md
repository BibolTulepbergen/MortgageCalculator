# Решение проблемы HTTP 404/502 при сборке APK

## Проблема

При сборке APK в GitHub Actions возникают ошибки:
- **HTTP 404: Not Found** — Ресурсы недоступны по ожидаемым адресам
- **HTTP 502: Bad Gateway** — Удаленные серверы временно недоступны

## Реализованное решение

### 1. Механизм повторных попыток (Retry Logic)

✅ **Файл:** `.github/workflows/android.yml`

Каждая попытка сборки теперь делается 3 раза с ожиданием 60 секунд между попытками:

```bash
for attempt in {1..3}; do
  if buildozer android debug; then
    break
  fi
  sleep 60  # Ожидание перед повтором
done
```

**Преимущества:**
- Обрабатывает временные сетевые сбои
- Дает время серверам восстановиться
- Значительно повышает успешность сборки

### 2. Оптимизация Gradle

✅ **Файл:** `gradle.properties`

Добавлены параметры для улучшения обработки сетевых соединений:

```properties
systemProp.http.keepAlive=true
systemProp.http.maxConnections=5
systemProp.http.connectionTimeout=30000
systemProp.http.socketTimeout=60000
```

### 3. Конфигурация PIP

✅ **Файл:** `pip.conf`

Настроены повторные попытки для pip:

```ini
[global]
timeout = 120
retries = 5

[install]
retries = 5
```

### 4. Параметры Buildozer

✅ **Файлы:** `buildozer_32.spec`, `buildozer_64.spec`

Добавлены параметры:
- `warn_on_root = 1` — Улучшенное логирование
- `org.gradle.parallel=true` — Параллельная сборка
- `log_level = 2` — Подробное логирование

### 5. Глобальная конфигурация Buildozer

✅ **Файл:** `.buildozerrc`

Конфигурация на уровне пользователя для консистентности.

### 6. Улучшенная обработка APK

✅ **Workflow изменение:**

- Вместо поиска APK по всей системе (`find /`), теперь используется точный путь
- Добавлена проверка наличия файла перед копированием
- Добавлено логирование всех шагов

## Как это работает

### Процесс сборки с повторами

1. **Попытка 1:**
   - Если успешно → сборка завершена ✓
   - Если ошибка → ожидание 60 секунд

2. **Попытка 2:**
   - Повтор сборки с временно восстановившимися серверами
   - Если успешно → завершено ✓
   - Если ошибка → ожидание 60 секунд

3. **Попытка 3:**
   - Последняя попытка
   - Если не удалась → вывод логов и отказ

### Обработка HTTP ошибок

```
HTTP 502 Bad Gateway
        ↓
Ожидание 60 сек (серверы восстанавливаются)
        ↓
Повторная попытка → Обычно успешно ✓
```

## Если проблема все еще есть

1. **Проверьте интернет соединение:**
   ```bash
   ping google.com
   ```

2. **Увеличьте количество повторов:**
   - Измените `max_attempts=3` на `max_attempts=5` в workflow

3. **Увеличьте время ожидания:**
   - Измените `sleep 60` на `sleep 120` (2 минуты)

4. **Проверьте логи сборки:**
   - В GitHub Actions смотрите полный лог шага "Build X-bit APK"
